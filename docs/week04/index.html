Beyond Arrays

Arrays and lists are wonderful things. If you need to keep several things of the same type together so you can work with them, an array or list is the way to go. But what about more complicated data? Bugs will creap into this code and the application it is part of will fail. Thankfully, the creators of C provided a way out of this mess...the struct.

Structs - keeping related data together

Structs allow us to mingle different types of data together, and build relationships between the data. Using the medication example, you could create a struct called Medication that holdss the medication's name, its manufacturer, its best-by date, and the amount of the medication on hand, all in one place for each medication. Then you can have a whole bunch of these medications in an array.

When you make a struct, you are creating a new type of thing in the C language that has never existed before. Now in addition to int, double, and char, you have the type medication. You can use it just like any of the other types...paramters, variables, return types, etc.
